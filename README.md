# LLM4CPpaper-rp

Replication package for the paper:

**A Study on Leveraging Large Language Models for Category-Partition Testing**

---

## 🔗 Zenodo Dataset

The experimental dataset is available at:

[Replication Package: A Study on Leveraging Large Language Models for Category-Partition Testing](https://doi.org/10.5281/zenodo.20141514)

---

## 📌 Description

This repository contains the code and scripts used to reproduce the experimental pipeline for LLM-based Category-Partition Method (CPM) test generation.

It includes:
- CPM-based test case generation via prompt engineering
- Direct test case generation using unified prompting strategy
- Evaluation of LLM-generated outputs

---

## 📁 Repository Structure

### 1. Python Project (`src/`)
Contains scripts for evaluation and visualization of experimental results:
- `BarChart.py`: generates a comparative bar chart showing average LLM evaluation scores (ChatGPT, Gemini, DeepSeek) across the five qualitative criteria (Correctness, Completeness, Coverage, Clarity, Consistency).
- `HeatMapChatGPT.py`: produces a heatmap visualization of ChatGPT performance across evaluation dimensions and test cases.
- `HeatMapGemini.py`: generates heatmap-based analysis for Gemini outputs using the same evaluation criteria.
- `HeatMapDeepSeek.py`: generates heatmap-based analysis for DeepSeek outputs to support cross-model comparison.
- `LinePlot.py`: produces line-based visualizations to analyze trends and variations in evaluation scores across criteria or experimental runs.

---

### 2. Inputs (`inputs/`)
This folder contains the input artifacts used to drive the experimental pipeline and support the CPM-based test generation process:
- `requirements_full.csv`: complete set of functional requirements describing the e-learning platform used as the experimental subject. These requirements cover both core and extended system functionalities.
- `requirements_eval.csv`: subset of five representative requirements selected for the qualitative evaluation of LLM-generated test artifacts and manual CPM baseline comparison.
- `use_case_description.txt`: detailed description of the e-learning platform used in the study. The system simulates a realistic educational environment supporting both synchronous and asynchronous teaching activities, including course management, live sessions, and user interaction features.

---

### 3. Prompts (`prompts/`)
This folder contains the prompt template used in the study to apply the Category Partition Method (CPM) and generate test cases from functional specifications:
- `cp_prompt.txt`: unified prompt used to guide the LLM in identifying input parameters, deriving categories and choices, and generating test cases directly from functional specifications.

---

### 4. Data (`data/`)
This folder contains all artifacts generated during the experimental study, including LLM outputs and evaluation materials.

The structure is organized as follows:
- `evaluation/`
  Contains all figures and visualizations produced during the analysis phase, including bar charts, heatmaps, and line plots used to compare the performance of ChatGPT, Gemini, and DeepSeek across the defined evaluation criteria.
- `results_chatgpt/`
  Contains the raw outputs generated by ChatGPT for each evaluated requirement. Each file includes both the CPM structures (category-partition specifications) and the corresponding generated test cases in a single consolidated document (PDF format).
- `results_deepseek/`
  Contains the same type of outputs generated by DeepSeek, maintaining consistency with the other models for comparative evaluation purposes.
- `results_gemini/`
  Contains the same type of outputs generated by Gemini, following the same structure as ChatGPT (CPM + test cases in a single file per requirement).

---

## 🧪 Experimental Subject

The experimental evaluation is conducted on an e-learning platform modeled through a set of functional requirements covering user registration, course management, class scheduling, and synchronous interaction features.

These requirements serve as the basis for evaluating the effectiveness of LLMs in generating CPM-based test artifacts.

---

## ▶️ Replication Procedure

To reproduce the experiments, run the following steps:

```bash
python src/BarChart.py
python src/HeatMapChatGPT.py
python src/HeatMapGemini.py
python src/HeatMapDeepSeek.py
python src/LinePlot.py 
```
